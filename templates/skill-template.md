# Skill 开发模板

## Skill 文件结构

```
~/.claude/skills/{skill-name}/
├── SKILL.md           # 主 skill 文件（必须）
├── package.json       # 元数据（推荐）
├── README.md          # 说明文档（可选）
└── templates/         # 支持文件（可选）
    └── ...
```

或简单形式（单文件 skill）：
```
.claude/skills/{skill-name}.md
```

## 设计指南

### 好的 Skill 特征
1. **单一职责**：一个 skill 解决一类问题
2. **自包含**：包含执行所需的全部知识
3. **可验证**：每个阶段有明确的完成标准
4. **有修复循环**：失败时有自动重试机制（建议最多 3 轮）
5. **有时间预估**：每个阶段标注预期耗时

### 避免的陷阱
1. **过于通用**：skill 应该比 CLAUDE.md 更具体
2. **缺少验证**：没有验证步骤的 skill 容易产出半成品
3. **硬编码路径**：使用相对路径或变量，保持可移植性
4. **忽略错误**：必须定义错误处理策略

### description 字段写作技巧
description 决定了 skill 是否会被正确触发：
- 包含功能描述 + 触发场景 + 关键词
- 中英文都写（如果是双语项目）
- 越具体越好，避免泛泛描述
