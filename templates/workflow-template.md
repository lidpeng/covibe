# 开发工作流模板 (Dev Workflow Template)

> 基于 Plan → Implement → Verify → Fix 循环，适配不同项目类型。

## 通用工作流骨架

```markdown
# {项目名} 自动化开发工作流

当用户提出需求时，按以下流程自动执行。

## 触发条件
用户描述了功能需求、Bug 修复或改进请求。

## 执行流程

### Phase 1: 规划（30s 内完成）
1. 分析需求，判断影响范围
2. 用 TodoWrite 列出实施步骤（3-7 步，每步可独立验证）
3. 如果涉及 >5 文件或架构变更，先用 EnterPlanMode 确认

### Phase 2: 实现（逐步推进）
对每个子任务：
1. 读取相关代码，理解现有实现
2. 编写代码改动
3. 改完立即标记 TodoWrite 为 done
4. 如果完成一个可测试单元，立即 git commit

### Phase 3: 验证
```bash
{项目测试命令}
```
检查项：
- {检查项 1}
- {检查项 2}
- {检查项 3}

### Phase 4: 修复循环（最多 3 轮）
如果 Phase 3 发现问题：
1. 分析错误，定位根因
2. 修复代码
3. 重新验证
4. 3 轮后仍有失败，停下来报告

### Phase 5: 收尾
1. 确认所有 TodoWrite 任务完成
2. {项目特定收尾操作}
3. 最终 git commit
4. 输出完成摘要

## 关键原则
- **不交付半成品**
- **不用假数据**
- **原子性提交**
- **最小改动**
```

---

## 项目类型适配

### Web 全栈项目（React + Python/Node）

追加 **Phase 4b: UI 视觉验证**：
```markdown
### Phase 4b: UI 视觉验证（前端改动必须执行）
1. 判断目标页面是否需要登录
2. 用截图工具截取目标页面
3. 用 Read 工具查看截图，判断渲染是否正确
4. 问题则修复后重新截图，最多 3 轮
5. 通过后才能 commit
```

验证命令：
```bash
# 后端
{后端测试命令}

# 前端构建
{前端构建命令}

# API 健康检查
curl -s http://localhost:{端口}/api/health

# UI 截图
{截图命令}
```

### 纯后端 API 项目

追加 **API 合约验证**：
```markdown
### 验证扩展: API 合约
1. 运行 API 测试套件
2. 检查响应 schema 是否符合约定
3. 验证错误码和错误消息
```

### 移动端项目

追加 **多平台验证**：
```markdown
### 验证扩展: 多平台
1. iOS 模拟器验证
2. Android 模拟器验证
3. 检查平台特定的 UI 差异
```

### CLI 工具项目

追加 **命令行验证**：
```markdown
### 验证扩展: CLI
1. 运行 --help 检查命令注册
2. 执行核心命令验证功能
3. 检查 exit code 和 stderr 输出
```

## 提交规范模板

```markdown
### Git 规范
- 前缀: feat / fix / docs / chore / refactor / test
- 格式: `{前缀}: {简要中文描述}`
- 每个子任务完成后立即 commit
- 不合并不相关的改动
- 不自动 push（需用户指示）
```
