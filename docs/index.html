为什么需要 covibe

多人 Vibe Coding 的协作困境

AI 编程助手让每个人效率翻倍，但团队协作的问题反而更严重了。

😰 没有 covibe

  • Alice 让 AI 删了手机号验证，Bob 不知道又加回来
  • Bob 踩坑花 3 小时，下周 Charlie 又踩一遍
  • 新人打开项目，AI 一头雾水
  • 两人同时改一个文件，合并时一堆冲突
  • 每个人的 AI 配置不一样，质量参差不齐

🎵 有 covibe

  • 编辑同一文件时实时预警，避免冲突
  • 踩坑经验自动记录，自动提醒下一个人
  • 新人一条命令就能上手整个项目
  • 团队规则自动同步到每个人的 AI
  • AI 分析代码库和成员画像，智能分工

全部功能

为 Vibe Coding 打造的完整工具链

从一个人到一个团队，从初始化到持续协作。

🔍

一键初始化

自动探测技术栈（Node/Python/Go/Rust/全栈），生成 CLAUDE.md + 配置 + 工作流。
covibe init

📊

100 分制审计

14 项检查覆盖知识、工具、权限、记忆、质量、团队六个维度。
covibe audit

🔗

P2P 实时同步

团队任一成员电脑上启动 sync server，零额外基建。局域网 WebSocket 实时同步。
covibe sync start

📋

共同任务看板

团队共享看板，每个人的 AI 都能看到、更新。
covibe board add "任务"

💡

共享经验板

零基建的团队知识库，踩过的坑自动提醒下一个人。支持点赞和自动归档。
covibe experience add

🎯

Agent 智能分工

基于 git blame + 成员画像，AI 推荐最优任务分配方案。
covibe coordinate "任务"

👤

自动成员画像

从 git 历史分析技能、活跃模块、工作时间，一键生成。
covibe profile-gen

🔑

Token 认证

团队初始化自动生成 token，sync 连接自动校验。通过 Git 分发，无需手动配置。

💌

团队建议系统

给队友发建议，接收方 AI 判断相关性，由接收者决定是否采纳。发送自由，接收自主。
covibe suggest send bob "建议"

🌐

跨 IDE 支持

从 CLAUDE.md 自动生成 Cursor Rules 和 Windsurf Rules。
covibe cross-ide all

理论基础 / Foundation

Harness Engineering 工作台工程

covibe 不只是协作工具——它建立在一套完整的 AI Agent 工作环境方法论之上。

🧠 核心理念

  • Harness ≠ 智能，Harness = 释放智能的工具链
  • 同一个模型，差的 harness 平庸，好的 harness 专家级
  • 大多数 AI Agent 失败是 harness 失败，不是模型失败

📐 三轴模型

  • 工具轴 — MCP、Shell、CLI → 赋予行动能力
  • 知识轴 — CLAUDE.md、经验板、记忆 → 赋予认知能力
  • 权限轴 — Hooks、团队规范 → 约束行为边界

IDE 原生集成

在你喜欢的 IDE 中直接使用

不用切换终端。covibe 原生集成到 Claude Code、Cursor、Windsurf。

💻

Claude Code

作为原生 Skill 集成，对话中直接输入：

/covibe init
/covibe audit
/covibe board add "任务"

AI 自动调用 CLI，无需切换终端。

🖱️

Cursor

一键从 CLAUDE.md 生成 Cursor Rules：

covibe cross-ide cursor
# → .cursorrules
# → .cursor/rules/covibe.md

团队规范自动同步到 Cursor AI。

🏄

Windsurf

同样一键生成 Windsurf Rules：

covibe cross-ide windsurf
# → .windsurfrules

团队成员用不同 IDE？规则保持一致。

快速开始

30 秒上手

# 安装
npm install -g covibe

# 在你的项目中
cd your-project
covibe init           # 自动探测 → 一键生成完整 AI 工作环境
covibe audit          # 看看健康度几分

# 开启团队模式
covibe team init      # 创建团队配置 + 生成 token
covibe sync start     # 启动实时同步
covibe dashboard      # 打开 Web Dashboard 看像素小人

# 日常使用
covibe board add "实现用户登录"
covibe board vibe 1
covibe experience add "Redis 连接池不超过 50" --category performance
